# AI Rules

## Working Style

Do not rewrite existing code unless necessary.

Do not refactor unrelated files.

Keep commits small.

Implement only one milestone at a time.

Wait for approval before moving to the next milestone.

Always explain why a change is needed.

---

## Priority

Correctness

Maintainability

Readability

Extensibility

---

## Before Coding

Read all markdown documentation.

Read existing shell script.

Understand business flow.

Never assume missing requirements.

Ask questions when uncertain.

---

## Coding

Prefer small reusable functions.

Avoid duplicated code.

Avoid hardcoded values.

Use config.sh.

Use meaningful variable names.

Prefer defensive programming.

---

## Testing

Each milestone must be executable independently.

Do not continue if previous milestone fails.

---

## Documentation

Update README when adding new feature.

Document assumptions.

Document limitations.
